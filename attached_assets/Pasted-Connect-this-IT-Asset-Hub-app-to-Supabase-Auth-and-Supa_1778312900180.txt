Login requirements:
1. Replace current frontend-only login with Supabase email/password login.
2. Use supabase.auth.signInWithPassword().
3. Email field should accept real user email.
4. Password field should accept real password.
5. Show loading state while login is processing.
6. Show clear error message if login fails.
7. After login, fetch user profile from public.profiles table.
8. Use the profile role to redirect the user.
9. If profile does not exist, show:
   “Your account profile is not configured. Please contact IT Admin.”
10. If profile status is inactive, show:
   “Your account is inactive. Please contact IT Admin.”
11. Do not allow inactive users to access the app.

Roles:
- super_admin
- it_admin
- it_agent
- end_user

Role access:
1. super_admin can access everything.
2. it_admin can manage assets, assignments, returns, tickets, reports, and users except Super Admin deletion.
3. it_agent can access assigned tickets, open unassigned tickets, and asset handover/return tasks.
4. end_user can only access their own assets, raise tickets, and view their own tickets.
